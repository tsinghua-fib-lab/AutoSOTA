## What Worked
1. **Increasing norm_const from 0.4 to 0.6**: The default 0.4 was slightly too small, causing values to occupy a narrow pixel range in the 224x224 image. Scaling to 0.6 better utilized the pixel dynamic range for the MAE reconstruction.
2. **Multi-scale ensembling**: Averaging predictions from 3 context lengths (2880, 1440, 720) reduces variance. Different temporal scales capture complementary patterns — the long window provides seasonal context while the short window captures recent dynamics.
3. **Weighted ensemble (0.6/0.25/0.15)**: Emphasizing the longest context slightly outperforms equal or uniform weighting, as the long context contains more reliable seasonal information for ETTh1.

## What Didn't Work
- **norm_const < 0.6** (e.g., 0.3, 0.35): Values mapped to even smaller pixel range; information compressed.
- **norm_const > 0.6** (e.g., 0.65, 0.7): Too much deviation from original training distribution of MAE; reconstruction degrades.
- **align_const tuning** (0.35, 0.45): Default 0.4 is already optimal for ETTh1.
- **Reducing seq_len alone**: While it helped when combined in ensemble, single smaller seq_len was not tried alone (expected to be suboptimal).

**State-of-the-Art Techniques**

- **Multi-Modal Representations:** Encoding time series as images (and even as text) is now seen as highly effective. VisionTS itself and follow-ups (e.g. DMMV, Time-VLM) all use image-based representations and often combine additional views. For example, Ni *et al.* (IJCAI 2025) survey highlight that vision-based encodings can capture patterns that LLMs miss (www.ijcai.org). Modern practice often includes creating multiple “modalities” of the series (e.g. raw image, season-trend decomposition, or textual summary) and fusing them into the forecaster (www.researchgate.net) (proceedings.mlr.press). 
- **Pre-trained Foundation Models:** Using large pre-trained models (from images or language) and adapting them to TSF is common. Besides VisionTS’s use of MAE (ImageNet-pretrained ViT), current best practice includes large TS-specific pretraining (e.g. Google’s TimesFM) or leveraging VLMs. For instance, Time-VLM uses frozen VLMs (vision+language) to inject rich semantic and temporal features (proceedings.mlr.press), aligning with the trend of “foundation models for TSF.” Fine-tuning even a bit (or zero-shot) with such models often yields strong gains. 
- **Data Augmentation:** Recent works apply creative augmentations to time series, especially via their image forms. These include *time warping, jittering, scaling,* or even *mixing* of different series. In the vision context, one can also apply image augmentations (random crops, slight shifts or noise) to the encoded images to create ensembles. Author recommendations (from surveys) emphasize careful normalization and consistent transformations to maintain seasonality. In practice, augmentations like random cropping of windows, Gaussian noise injection, or patch shuffling have been used to improve robustness. 
- **Ensembling and Multi-Horizon Fusion:** Best systems ensemble multiple forecasts. For example, one can ensemble predictions across different context windows or mask configurations. VisionTS itself averages across four prediction lengths; similarly, you might run multiple forward passes with different random masks or data augmentations and average. More broadly, *model ensembling* — combining VisionTS outputs with those of specialized TS models (like N-HiTS/TimesNet/Autoformer) — is standard to boost accuracy. Weighted averaging or stacking (via a simple linear regression on outputs) is often employed to squeeze out extra performance. 
- **Multi-Quantile and Probabilistic Forecasting:** Instead of only point forecasts (mean), state-of-art models output multiple quantiles or full distributions. VisionTS⁺⁺ introduces multiple output heads for quantiles (openreview.net), which is becoming common. Calibrated quantile forecasts allow more nuanced error measures and often improve (average) point accuracy after proper aggregation. In practice, models may predict e.g. the 10th, 50th, and 90th percentile concurrently. This does not hurt point MAE/MSE and can aid in capturing uncertainty. 
- **Post-Processing:** Recent work highlights the importance of careful post-processing. For example, forecasted values are often re-scaled or smoothed based on known seasonality. A common trick is to compute a short-window moving average or apply STL decomposition to the forecast residuals for correction. Another practice is to enforce invariances (e.g. known zeros remain zero). Surveys note that since the vision models were trained on images, one must undo any “patch norm” artifacts and map pixel outputs back to properly scaled time values. Ensuring outputs respect range/units (e.g. applying the same normalization as inputs) is critical.
